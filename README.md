- `analysis/` contains the report Rmarkdown file `HW4_final_report.Rmd`, the .pdf output of knitting the file `HW4_final_report.pdf`, and `preamble.tex` which contains additional LaTeX commands used in the report. Note that, while not included in this repo, a cache directory will be created in `analysis/HW4_final_report_cache` when knitting the .Rmd file.
- `logs/` contains the .out stdout and .err error output from the Slurm jobs for the simulation study. The filenames have the following format where `${SLURM_ARRAY_TASK_ID}` is the iteration (same as the `i`): `n${n}_sim_${SLURM_JOB_ID}_${SLURM_ARRAY_TASK_ID}.${ext}`. This repo contains only the log files from the 1st iteration for each sample size array job as an example.
- `results/` contains .pdf plot images generated by knitting the `analysis/HW4_final_report.Rmd` file.
- `sim/` contains `sim_data/` and `sim_out/` as well as the final, concatenated output of the results in `./sim/sim_out`. These are the final results of the simulation study with 3 different sample sizes: `sim_n100.txt`, `sim_n100.txt`, and `sim_n10000.txt`
- `sim/sim_data/` contains the .Rds output of jobs. This repo contains only the output for the 1st iteration for each sample size array job as an example: `sim_n100_1.Rds`, `sim_n1000_1.Rds`, and `sim_n10000_1.Rds`
- `sim/sim_out/` contains the .txt estimates output of jobs. This repo contains only the output for the 1st iteration for each sample size array job as an example: `sim_n100_i1.txt`, `sim_n1000_i1.txt`, and `sim_n10000_i1.txt`
- `source/` contains shell and R-scripts used for this report. The `cavi_functions.R` file contains functions implementing a Coordinate Ascent Variational Inference (CAVI) algorithm for a Bayesian K-Mixture Model of unit-variance univariate Gaussians. The `gibbs_functions.R` file contains scripts implementing a Gibbs Sampling algorithm for the same Bayesian model. The `utility.R` file contains functions for reordering results and assessing computation time. The `simulate_data.R` contains a  function to generate data for the simulation study. The `sbatch_run_sim.sh` shell script is submitted to Slurm to run the simulation as an array job; it loads R, creates necessary directories/initiates the `sim_n${n}.txt` file with a header then runs the simulation by calling `run_sim.R`.  Given `i` and `n` arguments, `run_sim.R` runs a single simulation for the ith iteration with sample size n and saves output to `sim/sim_data/` and `sim/sim_out/`.

- here

```bash
# load the R module
module load R/4.5.2

# install the
Rscript -e 'install.packages("here")'

# unload the R module
module unload R/4.5.2
```

The `sbatch_run_sim.sh` script assumes that the cluster is using the Slurm system for job scheduling and that [Environment Modules](https://modules.readthedocs.io/en/latest/index.html) is installed on the cluster with module `R/4.5.2` available to load. Different package management configurations (ie, spack, etc.) will require modifying the `sbatch_run_sim.sh` file to properly load R. Different cluster schedulers (ie, SGE, etc.) will require more extensive modifications to `sbatch_run_sim.sh` and to the submission command to run the simulation. The `here` package should be installed in the user's R package library directory.


### Example Slurm Command on RHPC

The simulation study was run on the RHPC cluster. 

Below is the an example of shell commands to submit 3 (one for each *n* in {100, 1000, 10000}) array jobs to Slurm. The script uses 1 cpu per task. The default array specification in `sbatch_run_sim.sh` is 1-500 (for nsim=500 simulations) with a limit of a maximum of 100 concurrent tasks per job. As a task in an array, each simulation *i* job has a unique jobid of the format `${SLURM_JOB_ID}_${SLURM_ARRAY_TASK_ID}`; the `${SLURM_JOB_ID}` value is shared with all other simulations for sample size *n*.

```bash
# project directory
pdir=full/path/to/bios731_hw4_parrish

# script-make sure it is set to executable!
script=${pdir}/source/sbatch_run_sim.sh

# submit array job for each n
for n in 100 1000 10000; do
	# submission command
	submission_command='sbatch --job-name=n'${n}'_sim --export=ALL,n='${n}' --chdir='${pdir}'/logs --output='${pdir}'/logs/%x_%A_%a.out --error='${pdir}'/logs/%x_%A_%a.err '${script}

	# print full command
	echo ${submission_command}

	# submit command and get jobid
	jobid=`${submission_command} | cut -d" " -f4`

	echo ${jobid}" submitted"

done
```

Log and error files will be found in `./logs/`. Example files are provided for the 1st iteration of my simulation.

A file for each sample size *n* and simulation *i* is saved as `./sim/sim_data/sim_n${n}_${i}.Rds` and can be copied from the cluster to a local folder using `scp`. An example command is shown below:

```bash 
scp user@cluster:/full/path/to/bios731_hw4_parrish/sim/sim_data/sim_n${n}_${i}.Rds /local/full/path/to/bios731_hw4_parrish/sim/sim_data/
```

Output for the 1st iteration from my simulation are `./sim/sim_data/sim_n${n}_1.Rds` and are available in this repo.

Once results are finished, concatenate the estimates results using:

```bash
# loop over sample sizes
for n in 100 1000 10000; do
	# loop through simulations
	for i in {1..500}; do
		cat ./sim/sim_out/sim_n${n}_i${i}.txt >> ./sim/sim_n${n}.txt
	done
done
```

The concatenated estimate results from my simulation are found in the `./sim/sim_n${n}.txt` files available in this repo.
